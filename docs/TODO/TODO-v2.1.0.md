## 2.1.0 版本目标

### 1. Python 解析逻辑迁移（核心）

- [ ] 将 Python 中的解析逻辑逐步迁移到 Rust AST
- [ ] Rust 作为唯一解析核心
- [ ] Python 仅保留作为 UI / 运行时桥接层（glue）
- [ ] 禁止 Python 和 Rust 同时维护两套解析逻辑（避免行为分裂）

> 目标：统一解析行为，而不是简单搬代码

---

### 2. 影响体验的 bug 修复

#### 空行问题（反复解析）

- [ ] 修复多次解析导致的空行累积问题
- [ ] 统一换行与段落规范，保证 parse/render 行为一致

---

#### 删除“玄武岩”徽标

- [ ] 移除玄武岩相关组件
- [ ] 清理对应渲染逻辑与样式残留

> 注意：这是功能清理，不属于 bug 修复

---

### 3. UI 优化

- [ ] 优化 WYSIWYG 编辑器渲染一致性
- [ ] 减少解析结果与界面展示之间的跳动
- [ ] 优化嵌套组件显示结构

---

### 4. 自定义 div / CSS 扫描器与 WYSIWYG 对齐

- [ ] 统一 div 扫描器输出结构
- [ ] 保证扫描结果与编辑器 DOM 表现一致
- [ ] 建立 AST → UI 映射规则
- [ ] 避免不同模块对同一结构的重复解释

---

## 总体目标总结

本版本核心目标：

> 将解析系统收敛为 Rust 单核实现，Python 仅作为 UI glue 层，并统一扫描器与 WYSIWYG 的语义模型，同时清理历史遗留组件。
